# تحديثات واجهة التطبيق - Mobile UI Updates

## التغييرات المنفذة ✅

### 1. الصفحة الرئيسية (HomeScreen)

#### الرسم البياني
- ✅ الرسم البياني الآن يملأ الشاشة بالكامل (60% من ارتفاع الشاشة)
- ✅ إزالة الحواف والظلال
- ✅ حذف زر تكبير الرسم البياني
- ✅ حذف شريط الأدوات العلوي والسفلي

#### كارت التداول التلقائي
- ✅ تحويله إلى كارت عائم صغير في أعلى الشاشة
- ✅ تغيير النص من "استقبال تحاليل" إلى "التداول التلقائي"
- ✅ تصميم مدمج مع نقطة حالة ملونة

#### أزرار التداول
- ✅ إضافة أزرار BUY و SELL أعلى الصفحة الرئيسية
- ✅ عرض حجم اللوت بين الزرين
- ✅ تصميم احترافي مع ألوان مميزة

### 2. صفحة السجلات (TradesScreen)

#### تصميم الصفقات المفتوحة
- ✅ تصميم مدمج يأخذ مساحة أقل
- ✅ عرض جميع المعلومات في صف واحد
- ✅ زر إغلاق صغير ومباشر

#### أزرار الإغلاق الجماعي
- ✅ إضافة 3 أزرار:
  - إغلاق المراكز الرابحة
  - إغلاق جميع مراكز البيع (SELL)
  - إغلاق جميع مراكز الشراء (BUY)

#### إزالة زر إعادة التعيين
- ✅ حذف زر "إعادة تعيين الحساب" من صفحة السجلات

### 3. نظام الهامش (Margin System)

#### تحديث الرافعة المالية
- ✅ تغيير الرافعة من 100 إلى 500
- ✅ حساب الهامش بناءً على الرافعة الجديدة

#### نظام Margin Call
- ✅ إغلاق تلقائي لجميع المراكز عند وصول الهامش إلى 20% من الرصيد
- ✅ حماية الحساب من الخسائر الكبيرة

#### حساب حجم الصفقة
- ✅ حجم الصفقة يعتمد على الـ equity (الرصيد + الأرباح العائمة)
- ✅ استخدام الرافعة المالية 500 في الحسابات
- ✅ التحقق من توفر الهامش قبل فتح الصفقة
- ✅ منع فتح صفقات أكبر من الرصيد المتاح

### 4. وظائف جديدة في paperTradingService

```typescript
// حساب حجم اللوت الأمثل
calculateOptimalLotSize(equity, marketPrice, riskPercent)

// إغلاق المراكز الرابحة فقط
closeProfitablePositions(marketPrice)

// إغلاق جميع مراكز الشراء
closeAllBuyPositions(marketPrice)

// إغلاق جميع مراكز البيع
closeAllSellPositions(marketPrice)
```

## الخطوة التالية

لتطبيق التحديثات على التطبيق:

```bash
cd mobile
npx eas update --branch production --message "تحديث واجهة المستخدم ونظام الهامش"
```

أو لبناء APK جديد:

```bash
npx eas build --platform android --profile production
```
